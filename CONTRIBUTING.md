# Contributing to BehaviorIQ ML Service

Thank you for your interest in contributing. This file provides a short pointer to get started — keep contributions small and focused.

How to contribute

1. Open an issue to describe the feature or bug you plan to work on.
2. Fork the repository and create a feature branch: `git checkout -b fix/your-short-description`.
3. Write tests where applicable and run them locally.
4. Open a pull request against `main` with a clear description of the change.

Code style

- Follow the existing project style. Use modern Python (3.10+) and type hints where appropriate.
- Run a formatter (black) and a linter before creating a PR.

Communication

- Use issues for design discussions. Keep PRs focused on implementation.
- Tag repository owner or reviewers in PRs for review.

Security

- Do not commit secrets or API keys. Use environment variables for keys (see README).

License

- Add a `LICENSE` file to the repo to make contribution/licensing terms explicit.
